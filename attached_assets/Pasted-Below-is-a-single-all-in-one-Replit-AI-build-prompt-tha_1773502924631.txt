# Replit Full Backend Build Prompt (America’s Rant Line)

Build the backend for a web platform called **America’s Rant Line**.

The platform allows users to call a phone number, leave a voicemail rant, and have that rant appear on the website after moderation.

Users can vote on rants and view leaderboards.

The platform has **three rant lines**:

• MAGA Line — Right (Conservative / Republican)
• Blue Line — Left (Democrat / Progressive)
• Neutral Line — Independent / Open Rant

Each rant must receive a **Rant ID number**.

The platform must capture **phone numbers and emails from callers**.

# Supabase Database Setup

Create table **callers**

```sql id="callers-table"
id uuid primary key default uuid_generate_v4()
phone text
email text
nickname text
created_at timestamp default now()
```

---

```sql id="rants-table"
id uuid primary key default uuid_generate_v4()
rant_number serial
caller_id uuid
category text
audio_url text
duration integer
votes integer default 0

approved boolean default false

```

# Twilio Recording Webhook

Create endpoint:

```http id="webhook"
POST /api/twilio/recording
```

Twilio sends:

```json id="twilio-payload"
RecordingUrl
From
RecordingDuration
Digits
```

Server steps:

1. Determine rant category from Digits
2. Download the recording from Twilio
3. Upload recording to Supabase Storage bucket `rant-audio`
4. Get public file URL
5. Check if caller phone exists in callers table
6. If not, create new caller
7. Insert rant record

Fields saved:

```id="rant-insert"
caller_id
category
audio_url
duration
approved = false
votes = 0
```
